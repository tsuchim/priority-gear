# PriorityGear v0.3.1 - GitHub installer release with adversarial installer gate

`v0.3.1` is the formal GitHub installer release built from the current post-PR #9 `main`.

## Summary

- Keeps the `PriorityGear.Setup.exe` installer distribution introduced in `v0.3.0`.
- Includes the adversarial installer state-machine gate added after `v0.3.0`.
- Publishes `PriorityGear-v0.3.1-win-x64-installer.zip` as the primary release artifact.
- Releases from the `main` state that includes PR #9.

## Installer

`PriorityGear.Setup.exe` is the normal install/update entrypoint. The installer installs the GUI app, CLI, and System Mode service payload, configures `PriorityGear.Service` as LocalSystem, and uses versioned install directories under `%ProgramFiles%\PriorityGear\versions`.

The installer preserves ProgramData rules and logs by default, including `%ProgramData%\PriorityGear\rules.machine.json` and `%ProgramData%\PriorityGear\Logs`. It also supports uninstall/cleanup behavior through:

```powershell
.\PriorityGear.Setup.exe --uninstall
```

## Adversarial installer gate

This release includes adversarial installer state-machine tests for:

- payload missing;
- service stop failure;
- partial copy failure;
- service configure failure;
- service start timeout;
- status pipe timeout;
- non-LocalSystem service account;
- stale or hostile service `binPath`;
- missing process identity;
- old version cleanup warning-only behavior;
- immutable result snapshots.

## Safety boundary

PriorityGear System Mode is privileged software and installs a LocalSystem Windows Service. It changes process priority and may affect system responsiveness or stability. It is provided AS IS.

The installer is unsigned. Store, winget, signing, MSI, and MSIX packaging are not included in this release.

This release does not expand unsafe service mutation support. Arbitrary shared-host `svchost.exe` priority mutation remains out of scope.

GitHub Actions builds and packages the installer. GitHub Actions does not run elevated setup.
