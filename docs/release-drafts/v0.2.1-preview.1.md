# PriorityGear v0.2.1-preview.1 - System Mode preview update

This is a small preview update after `v0.2.0-preview.1`. It is not the final stable `v0.2.0` release.

## Summary

- Uses the automated preview release workflow for preview tags.
- Improves operator-facing System Mode status visibility for already-supported service status data.
- Keeps the application icon wired into the WPF app and verification setup artifacts.
- Keeps the v0.2 System Mode safety boundary unchanged.

## Included

- System Mode status display includes service identity, service binary/version path, `SeDebugPrivilege`, machine-rule monitor summary, and service-process discovery truncation metadata.
- Release packaging continues to include the verification setup and payload executables.
- App icon configuration remains part of the WPF app executable and tray resource path.

## Safety boundary

PriorityGear System Mode is privileged software and installs a LocalSystem Windows Service during verification. It is provided AS IS and must be used at the user's own risk.

This preview does not expand unsafe service mutation support. Arbitrary shared-host `svchost.exe` priority mutation remains out of scope. Shared-host behavior remains guarded by service-name rules, administrator approval, and dry-run/reject safety.

## Out of scope

- Stable `v0.2.0` release.
- Production installer, MSI/MSIX, Store, winget, signing, or updater.
- GUI machine-rule editing.
- Arbitrary shared-host `svchost.exe` mutation.
- CPU affinity, I/O priority, EcoQoS, Realtime priority UI, drivers, telemetry, network features, or System Mode active-window switching.

## Validation

Expected validation for this preview:

```powershell
dotnet restore PriorityGear.slnx
dotnet build PriorityGear.slnx --configuration Release --no-restore
dotnet test PriorityGear.slnx --configuration Release --no-build
.\scripts\build-verification-installer.ps1
.\scripts\check-workflow-release-state.ps1
.\scripts\package-preview-release.ps1 -TagName "v0.2.1-preview.1" -SetupDirectory ".\artifacts\setup-v0.2\PriorityGear-v0.2-system-mode-verification" -OutputDirectory ".\artifacts\release-preview-test-v0.2.1"
.\scripts\inspect-release-artifacts.ps1 -ArtifactDirectory ".\artifacts\release-preview-test-v0.2.1" -TagName "v0.2.1-preview.1"
```

GitHub Actions does not run the elevated verification setup. Elevated setup verification remains a local, user-approved UAC operation.
