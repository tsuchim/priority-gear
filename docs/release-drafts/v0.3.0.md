# PriorityGear v0.3.0 - GitHub installer release

`v0.3.0` moves PriorityGear from verification-setup distribution to a formal GitHub release installer artifact.

## Summary

- Adds `PriorityGear.Setup.exe` as the normal install/update entrypoint.
- Publishes `PriorityGear-v0.3.0-win-x64-installer.zip` as the primary release artifact.
- Installs the GUI app, CLI, and System Mode service payload under a versioned Program Files directory.
- Configures `PriorityGear.Service` as LocalSystem and confirms service status through the status pipe.
- Adds `PriorityGear.Setup.exe --uninstall` for normal service cleanup while preserving ProgramData.

## Safety boundary

PriorityGear System Mode is privileged software and installs a LocalSystem Windows Service. It changes process priority and may affect system responsiveness or stability. It is provided AS IS.

The installer is unsigned. Store, winget, signing, MSI, and MSIX packaging are not included in this release.

This release does not expand unsafe service mutation support. Arbitrary shared-host `svchost.exe` priority mutation remains out of scope.

GitHub Actions builds and packages the installer. GitHub Actions does not run elevated setup.
